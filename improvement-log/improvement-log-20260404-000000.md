## Log

### 2026-04-04 — Initially hypothesized only the file watcher reload issue

**What happened:** After reading the code, I identified the self-triggered file watcher reload as the main issue. Only after building the app, running it, and reading the DebugTrace did I discover the actual primary cause: `atomically: true` writes fail under macOS sandbox for iCloud Drive directories (NSCocoaErrorDomain Code=513).
**Why this matters:** Code reading alone would not have found this. The `atomically: true` pattern is standard Swift and looks correct. The failure is a runtime macOS sandbox behavior.
**What better looks like:** For macOS sandboxed app bugs, always build and run the app first and check DebugTrace/logs before spending time on code-only analysis. Runtime behavior is the ground truth.
