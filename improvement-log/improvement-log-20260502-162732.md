## Log

### 2026-05-02 — macOS frontmatter scope was missing `deleteFrontmatterField`

**What happened:** When wiring up the metadata block fixes, I tried to build `Noto-macOS` and got `cannot find 'deleteFrontmatterField' in scope` at TextKit2EditorView.swift:6634. The function had only been defined inside the `#if os(iOS)` block, but the macOS click handler was already calling it.
**Why this matters:** The branch had been silently broken on macOS — the iOS-only definition never triggered a compile error in iOS builds, so it shipped to a worktree with a non-building macOS target.
**What better looks like:** When adding shared logic touched by both platforms, smoke-build both schemes (`flowdeck build … Noto-iOS` AND `Noto-macOS`) before assuming the change is complete. Treat shared call sites as a checklist.

### 2026-05-02 — iOS metadata block surface was too dark to perceive

**What happened:** The block used `0x080809`, a near-black gray. Against the editor's `0x0A0A0A` background, it disappeared visually. Also, it was being inserted as the bottom-most subview of `UITextView`, so the document body text was rendering on top of it.
**Why this matters:** The user's first complaint was that the block looked "too transparent" — both effects (low contrast and text bleed-through) compounded.
**What better looks like:** When picking a "card on a dark surface" gray, contrast it to a known step like `secondarySystemBackground` (≈ `0x1C1C1E`). And remember UITextView renders its text via the view's own draw pass, so subviews stay BEHIND the text unless they're added with `addSubview` to the top of the stack.
