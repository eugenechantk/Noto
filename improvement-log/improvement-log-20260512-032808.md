### 2026-05-12 — Initial misreading of "sidebar goes empty"

**What happened:** When the user reported "sidebar goes empty after note loads", I first hypothesized that the new `expandToSelectedNote()` was working correctly but the user perceived the folder navigation as "empty" (because root content disappeared). Only after building and screenshotting did I see the literal "Empty" placeholder despite the folder containing 14 notes.
**Why this was wrong:** The user described the symptom literally ("empty"). I should have trusted the description and reproduced visually first, not theorized about perception. The actual bug — a SwiftUI `.task(id:)` not re-firing on store replacement — required a runtime repro to surface; static reading would not have caught it.
**What better looks like:** For UI bugs, build → reproduce visually → only then theorize. Especially when the words "empty", "blank", "missing", or "gone" appear, take them literally before reaching for perception-based explanations.
